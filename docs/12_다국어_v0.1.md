## 6. AI 응답 다국어

### 6.1 흐름

```
사용자 입력 (한/영/일 자동 감지)
    ↓
classifyInput → language: 'ko' | 'en' | 'ja'
    ↓
generateDraft 호출 시 language 전달
    ↓
시스템 프롬프트에 응답 언어 명시
    ↓
AI가 해당 언어로 응답
```

### 6.2 시스템 프롬프트 변형

```
당신은 ... AI입니다.

[응답 언어]
사용자 입력 언어: {language}
모든 응답을 {language}로 작성하세요.
- ko: 한국어 (존댓말)
- en: English (polite)
- ja: 日本語 (丁寧語)
```

### 6.3 사용자 입력 보존
- 사용자가 한국어로 입력 → 그대로 보존 (번역 X)
- AI가 만든 추가 정보(설명 등)만 응답 언어로

## 8. 시간 / 숫자 포맷

### 8.1 date-fns + locale

```typescript
import { format } from 'date-fns';
import { ko, enUS, ja } from 'date-fns/locale';

const localeMap = { ko, en: enUS, ja };

format(date, 'PPP', { locale: localeMap[currentLanguage] });
// ko: "2026년 4월 27일 월요일"
// en: "Monday, April 27th, 2026"
// ja: "2026年4月27日 月曜日"
```

### 8.2 숫자 포맷

```typescript
new Intl.NumberFormat('ko-KR').format(1234567);    // "1,234,567"
new Intl.NumberFormat('en-US').format(1234567);    // "1,234,567"
new Intl.NumberFormat('ja-JP').format(1234567);    // "1,234,567"
```

### 8.3 통화

```typescript
new Intl.NumberFormat('ko-KR', { style: 'currency', currency: 'KRW' }).format(13900);
// "₩13,900"
```

## 10. 푸시 알림 다국어

```typescript
};

await sendPushNotification(uid, message[user.language]);
```

## 12. 품질 관리

### 12.1 번역 누락 감지
- ESLint 룰: 하드코딩 한국어 텍스트 감지
- CI에서 모든 언어 키 일치 검증 (자동화)

### 12.2 번역 예산
- AI 1차 번역 → 사람 검수 (Beta 1 종료 후)
- 영어/일본어 모국어 화자 피드백

### 12.3 다국어 사용자 지표
- 언어별 사용자 비율 모니터링
- 일본어 사용자 < 5% 시 우선순위 하향
