# 환경 변수 가이드

**버전:** v0.1
**작성일:** 2026-04-27

---

## 0. 개요

이 문서는 본 프로젝트의 환경 변수 관리 방식을 정의한다.

**원칙:**
- 비밀 정보 절대 Git 커밋 X
- Production / Staging / Dev 분리
- 클라이언트 노출 변수와 서버 전용 변수 분리

---

## 1. 환경 분리

### 1.1 3개 환경

| 환경 | 용도 | Firebase 프로젝트 |
|---|---|---|
| **Production** | 실 사용자 | `app-prod` |
| **Staging** | 베타/QA | `app-staging` |
| **Dev** | 개발 | `app-dev` |

각 환경은 **별도의 Firebase 프로젝트** 사용 (데이터 격리, 비용 관리).

### 1.2 환경별 설정 파일

```
.env                          # 공통 + 로컬 dev (gitignored)
.env.example                  # 템플릿 (커밋)
.env.production               # CI/CD에서만 (gitignored)
.env.staging                  # CI/CD에서만 (gitignored)
```

## 2. 변수 분류

### 2.1 클라이언트 노출 (Public)

### 2.2 서버 전용 (Secret)
- Cloud Functions에서만 사용

### 5.2 Secret Manager 사용 (가장 민감한 키)

특히 민감한 키는 GCP Secret Manager 사용:

```typescript
import { defineSecret } from 'firebase-functions/params';

const geminiApiKey = defineSecret('GEMINI_API_KEY');
const stripeSecretKey = defineSecret('STRIPE_SECRET_KEY');

export const classifyInput = onCall(
  { secrets: [geminiApiKey] },
  async (request) => {
    const apiKey = geminiApiKey.value();
    // 사용
  }
);
```

## 7. CI/CD 환경 변수 설정

### 7.1 GitHub Actions

```yaml
# .github/workflows/mobile-build.yml
env:
  EXPO_PUBLIC_FIREBASE_API_KEY: ${{ secrets.PROD_FIREBASE_API_KEY }}
  EXPO_PUBLIC_FIREBASE_PROJECT_ID: ${{ secrets.PROD_FIREBASE_PROJECT_ID }}
  EAS_TOKEN: ${{ secrets.EAS_TOKEN }}
```

GitHub Secrets에 모든 production 변수 등록.

### 7.2 Vercel (어드민 콘솔)

Vercel 대시보드 → Settings → Environment Variables에서:
- Production / Preview / Development 별로 설정
- 민감 변수는 "Sensitive" 토글

### 7.3 Firebase Functions

```bash
# 배포 시 환경 변수 자동 사용
firebase deploy --only functions

# Secret Manager 시크릿 설정
firebase functions:secrets:set GEMINI_API_KEY
firebase functions:secrets:set STRIPE_SECRET_KEY
```

## 8. 로컬 개발 환경 셋업

### 8.1 첫 셋업

```bash
# 1. 저장소 클론
git clone ... && cd project

# 2. 의존성 설치
pnpm install

# 3. 환경 변수 복사
cp .env.example .env
# .env 편집 (개발자 본인 Firebase 프로젝트 키 입력)

# 4. Firebase Emulator 시작
firebase emulators:start

# 5. 모바일 앱 실행
pnpm mobile:dev

# 6. 어드민 콘솔 실행 (별도 터미널)
pnpm admin:dev
```

### 8.2 Firebase Emulator 활용
- 로컬 개발 시 실 Firebase 비용 0
- Auth, Firestore, Functions, Storage 모두 에뮬레이트
- Production 데이터 영향 X

## 9. 환경 변수 추가 가이드

새 환경 변수가 필요할 때:

1. **`.env.example` 업데이트** (모든 변수 명시)
2. **각 환경의 실제 값** 설정 (Vercel/GitHub Secrets/Firebase)
3. **이 문서 업데이트** (변수 목적 설명)
4. **타입 안전성 확보** (env validation 라이브러리, 예: `zod`)

```typescript
// shared/env.ts
import { z } from 'zod';

const envSchema = z.object({
  EXPO_PUBLIC_FIREBASE_API_KEY: z.string().min(1),
  EXPO_PUBLIC_FIREBASE_PROJECT_ID: z.string().min(1),
  EXPO_PUBLIC_ENV: z.enum(['development', 'staging', 'production']),
  // ...
});

export const env = envSchema.parse(process.env);
```

## 10. 주의사항

### 10.1 절대 금지
- ❌ `.env` 파일 Git 커밋
- ❌ 클라이언트에 서버 전용 키 노출 (`NEXT_PUBLIC_*` 접두 X)
- ❌ 패스워드/API 키 하드코딩
- ❌ Console log에 민감 정보 출력

### 10.2 권장 사항
- ✅ 모든 API 키 정기 회전 (3~6개월)
- ✅ 키 회전 시 변경 이력 기록
- ✅ 키 노출 의심 시 즉시 회전
- ✅ Pre-commit hook (gitleaks, truffleHog 등)으로 자동 검사
