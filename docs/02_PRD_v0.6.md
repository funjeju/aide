## 7. Phase 4 (정식 출시) — 협업/결제/안정화 3개 ⭐ NEW

### 🔹 FR-026: 협업 / 공유 기능

**Phase**: 정식 출시 | **우선순위**: HIGH | **AI 모델**: 사용 안 함

#### 사용자 스토리
> 사용자로서, 가족/팀원과 만다라트나 프로젝트를 공유하여 같이 목표를 추적하고, 비즈니스 리포트를 상사/동료에게 공유해 협업 효율을 높이고 싶다.

#### 7.1.1 공유 대상

| 대상 | 단위 | 비고 |
|---|---|---|
| **만다라트** | Project 전체 | 가장 일반적 (가족 비전 보드, 팀 OKR 등) |
| **만다라트 일부** | Section 단위 | 특정 영역만 공유 |
| **Task** | 개별 또는 그룹 | 위임/협업용 |
| **비즈니스 리포트** | 일/주/월간 | 상사 공유용 |
| **타임라인** | 일자 단위 | 가족/연인과 일기 공유 |

#### 7.1.2 권한 모델

| 권한 | 보기 | 수정 | 삭제 | 공유 (재공유) | 권한 변경 |
|---|---|---|---|---|---|
| **Owner** | ✅ | ✅ | ✅ | ✅ | ✅ |
| **Editor** | ✅ | ✅ | ❌ | ❌ | ❌ |
| **Commenter** | ✅ | ❌ (댓글만 가능) | ❌ | ❌ | ❌ |
| **Viewer** | ✅ | ❌ | ❌ | ❌ | ❌ |

`[⚠️ 결정 #D1]` 4단계 권한 vs 3단계 (Owner/Editor/Viewer만)

#### 7.1.3 공유 방식

**방식 A: 사용자 초대 (이메일)**
- 받는 사람이 앱 사용자여야 함
- 초대 → 푸시/이메일 알림 → 수락/거절
- 권한 지정 (Owner는 자동, 다른 권한 선택)

**방식 B: 공개 링크**
- 링크 생성 (읽기 전용 기본)
- 만료일 설정 (1일 / 7일 / 30일 / 무제한) `[⚠️ 결정 #D2]`
- 비밀번호 보호 (선택)
- 링크 폐기 가능

**방식 C: 그룹 (Pro 이상)**
- 사전 등록한 그룹에 일괄 공유
- 가족, 팀, 프로젝트 등으로 그룹 구성

#### 7.1.4 협업 인원 한도 (Tier별)

| Tier | 협업 인원 (1 항목당) | 그룹 |
|---|---|---|
| Free | 0명 (협업 불가) | - |
| Basic | 3명 | - |
| Pro | 10명 | 그룹 5개 |
| Enterprise | 무제한 | 무제한 |

#### 7.1.5 실시간 동기화
- Firestore Realtime Listener
- 다른 사용자 수정 → 즉시 반영
- 충돌 처리: Last-write-wins + 변경 알림
- 동시 편집 표시 (누가 지금 보고 있는지)

#### 7.1.6 협업 알림
- 항목에 새 댓글 추가
- Task 완료 (담당자가 누른 경우)
- 권한 변경
- 새로운 멤버 초대 수락
- 공유 항목 삭제

#### 7.1.7 데이터 소유권 `[⚠️ 결정 #D3]`
- **Owner가 떠나면**: Editor 중 가장 오래된 멤버에게 자동 이전 (추천)
- 또는: Owner 명시 지정 (떠나기 전 후임 지정)

#### 데이터 요구사항
- Firestore `shares/{shareId}`:
  - resourceType ('project' | 'section' | 'task' | 'report' | 'timeline')
  - resourceId, ownerId, sharedAt
- Firestore `shares/{shareId}/members/{userId}`:
  - role: 'owner' | 'editor' | 'commenter' | 'viewer'
  - invitedAt, acceptedAt, lastViewedAt
- Firestore `publicLinks/{linkId}`:
  - resourceType, resourceId, password (해시), expiresAt, createdBy

#### 인수 기준 (AC)
- [ ] 초대 → 수락 → 협업 시작 < 5분
- [ ] 실시간 동기화 지연 < 3초
- [ ] 권한 변경 즉시 반영
- [ ] 활성 협업 사용자 ≥ 100명 (정식 출시 게이트)
- [ ] 100명 동시 편집 지원

#### 엣지 케이스
- 협업자가 Free → Pro 강등 → 협업 항목 어떻게 처리:
  - 옵션: 보기만 가능 + 업그레이드 유도 (추천)
- 협업자가 계정 삭제 → 해당 사용자 데이터는 익명 처리
- 권한 없는 사용자가 공개 링크 통해 접근 → 회원가입 유도

### 🔹 FR-027: 결제 / 구독 모델

**Phase**: 정식 출시 | **우선순위**: HIGH | **AI 모델**: 사용 안 함

#### 사용자 스토리
> 사용자로서, 무료로 시작하지만 가치를 느끼면 합리적인 가격에 업그레이드해서 더 많은 한도와 기능을 이용하고 싶다.

#### 7.2.1 요금제 가격 (가정값) `[⚠️ 결정 #D4]`

| Tier | 월 요금 | 연 요금 (할인) | 연간 절약 |
|---|---|---|---|
| **Free** | $0 | - | - |
| **Basic** | $4.99 | $49.99 | $9.89 (16%) |
| **Pro** | $9.99 | $99.99 | $19.89 (16%) |
| **Enterprise** | $29.99 | $299.99 | $59.89 (16%) |

> 한국 시장 가격 별도 (₩6,900 / ₩13,900 / ₩39,900 정도)

#### 7.2.2 결제 인프라

**iOS**:
- Apple In-App Purchase + StoreKit 2
- Apple 30% 수수료 (1년 후 15%)

**Android**:
- Google Play Billing
- Google 30% 수수료 (1년 후 15%)

**구독 관리**: RevenueCat 사용 (강력 추천)
- iOS/Android 통합 처리
- 영수증 검증 자동
- 환불 처리 자동
- 분석 대시보드 제공

**웹 결제 (선택)** `[⚠️ 결정 #D5]`
- Stripe (App Store 외부 결제 + 수수료 없음)
- 단, 모바일 앱에서 외부 결제 유도는 Apple 정책 위반 위험
- Pro 이상에서만 옵션 제공 (직접 가입 페이지)

#### 7.2.3 구독 흐름

**구독 시작**:
1. 한도 초과 시 업그레이드 안내 → 요금제 화면
2. Tier 선택 → 월/년 선택 → 결제 (IAP)
3. 결제 성공 → Cloud Functions가 영수증 검증 → 사용자 Tier 업그레이드
4. 즉시 새 한도 적용

**자동 갱신**:
- 만료 24시간 전 자동 갱신 (IAP 기본 동작)
- 갱신 실패 → 7일 grace period → 자동 강등 (Free)

**업그레이드 (예: Basic → Pro)**:
- 일할 계산 차액 결제
- 즉시 새 한도 적용

**다운그레이드 (예: Pro → Basic)**:
- 다음 갱신 시점에 적용
- 현재 결제 기간 동안은 Pro 유지

**취소**:
- 자동 갱신 OFF
- 결제 기간 동안 유지 → 만료 후 Free 강등

#### 7.2.4 무료 체험 `[⚠️ 결정 #D6]`
- **추천**: 가입 후 14일 Pro 무료 체험
- 신용카드 등록 불필요 (이탈 방지)
- 만료 후 자동 Free 강등 (구독 안 하면)

#### 7.2.5 환불 정책 `[⚠️ 결정 #D7]`
- **추천 정책**:
  - 구매 후 7일 이내 사용 거의 없음 (음성 입력 < 5회 등) → 자동 환불 가능
  - 그 외 → 사례별 검토 (어드민 처리)
  - Apple/Google은 자체 환불 정책 있음 (사용자가 직접 요청)

#### 7.2.6 결제 모니터링 (어드민)
- MRR (Monthly Recurring Revenue)
- 신규 구독 수 / 취소 수 (churn)
- 평균 LTV (Lifetime Value)
- Tier별 분포

#### 7.2.7 결제 실패 처리
- 갱신 실패 → grace period 7일
- 사용자에게 알림 (앱 + 이메일)
- 결제 정보 업데이트 유도

#### 데이터 요구사항
- Firestore `subscriptions/{subscriptionId}`:
  - userId, tier, period ('monthly' | 'yearly'), startDate, endDate
  - autoRenew, status ('active' | 'canceled' | 'expired' | 'grace_period')
  - platform ('ios' | 'android' | 'web')
  - revenueCatId
- Firestore `payments/{paymentId}`:
  - userId, subscriptionId, amount, currency, paidAt, receiptId, status

#### 인수 기준 (AC)
- [ ] 결제 성공 → Tier 업그레이드 < 30초
- [ ] 영수증 검증 100% (위조 영수증 차단)
- [ ] 결제 전환율 ≥ 5% (정식 출시 게이트)
- [ ] 환불 처리 24시간 이내
- [ ] 취소율 (월) < 5%

#### 엣지 케이스
- 영수증 검증 실패 → 재시도 + 사용자 안내
- 한 사용자가 iOS와 Android 양쪽에서 구독 → 중복 알림 + 한쪽 취소 유도
- Apple/Google 환불 → Webhook 수신 → Tier 자동 강등
- 결제 페이지에서 한도 초과 안내 (예: Pro 한도 안내)

### 🔹 FR-028: 데이터 내보내기 (PDF, Excel)

**Phase**: 정식 출시 | **우선순위**: NORMAL | **AI 모델**: 사용 안 함

#### 사용자 스토리
> 사용자로서, 내 데이터를 PDF/Excel로 내보내서 백업하거나 보고용으로 활용하고, 필요 시 다른 도구로 마이그레이션하고 싶다.

#### 7.3.1 내보내기 대상 + 포맷

| 대상 | PDF | Excel/CSV | JSON |
|---|---|---|---|
| **만다라트 (Project)** | ✅ 시각적 | ✅ 표 형식 | ✅ 구조 |
| **Task 리스트** | ✅ | ✅ | ✅ |
| **비즈니스 리포트** | ✅ 시각적 | ✅ | ✅ |
| **타임라인 (일기)** | ✅ 일기 스타일 | - | ✅ |
| **전체 데이터** | - | - | ✅ (GDPR/PIPA) |

#### 7.3.2 PDF 템플릿
- 만다라트: 9x9 그리드 + 진행률
- 리포트: 차트 포함, 회사 보고서 스타일
- 일기: 다이어리 스타일

#### 7.3.3 Excel 구조
- 시트별 분리 (Tasks, Events, Persons 등)
- 정렬/필터 가능한 형식
- 헤더 스타일링

#### 7.3.4 JSON 구조 (전체 데이터)
- 사용자 데이터 모든 컬렉션 export
- GDPR/PIPA 데이터 이동권 대응
- 복원 가능한 형식

#### 7.3.5 내보내기 흐름
1. 설정 → "데이터 내보내기" 메뉴
2. 대상 선택 (만다라트/Task/리포트/타임라인/전체)
3. 기간 지정 (해당 시)
4. 포맷 선택
5. **즉시 다운로드** (작은 데이터) 또는 **이메일 발송** (큰 데이터)
6. 진행률 표시

#### 7.3.6 내보내기 한도

| Tier | 월 횟수 | 비고 |
|---|---|---|
| Free | 0회 | 비활성 |
| Basic | 3회 | |
| Pro | 무제한 | |
| Enterprise | 무제한 + API | API 접근 가능 |

#### 7.3.7 전체 데이터 내보내기 (GDPR/PIPA)
- 모든 사용자 (Free 포함) 권리
- 처리: 30일 이내 (법적 의무)
- 어드민 검토 후 발송

#### 기술 요구사항
- PDF 생성: React Native PDF 또는 서버 측 (puppeteer)
- Excel 생성: ExcelJS (서버 측, Cloud Functions)
- 큰 파일은 Cloud Storage 임시 저장 + 다운로드 링크 (24시간 유효)
- 비동기 처리 (큐) — 사용자 대기 시간 최소화

#### 데이터 요구사항
- Firestore `exports/{exportId}`:
  - userId, type, period, format, status, downloadUrl, requestedAt, completedAt, expiresAt

#### 인수 기준 (AC)
- [ ] 단일 만다라트 PDF 생성 < 5초
- [ ] 1년치 전체 데이터 JSON < 30초
- [ ] 다운로드 링크 24시간 유효
- [ ] 한도 초과 시 안내 + 업그레이드 유도

#### 엣지 케이스
- 매우 큰 데이터 (10GB+) → 분할 압축
- 생성 중 사용자 앱 종료 → 백그라운드 진행 + 완료 시 푸시 알림
- 내보내기 후 사용자가 데이터 삭제 → 다운로드 링크 즉시 무효

#### 🔹 FR-A04: Tier 정의 관리 (활성화)

**Phase**: Phase 4 | **우선순위**: HIGH

##### 핵심 동작
1. 어드민 콘솔 → "Tier 관리" 메뉴
2. 기존 Tier 목록 (Free / Basic / Pro / Enterprise)
3. Tier 생성 / 수정 / 비활성화:
   - 한도 (음성/텍스트/Project/협업)
   - 가격 (월/년)
   - 기능 활성화 (예: 데이터 내보내기 ON/OFF)

5. 적용:
   - 즉시 적용 (모든 사용자에게)
   - 신규 가입자만 적용 (기존 사용자 유지)
   - 점진 적용 (다음 갱신 시점)

##### 데이터
- `tiers/{tierCode}` 갱신
- 변경 이력 `admin_logs`

##### 인수 기준
- [ ] Tier 변경 즉시 모든 사용자 한도 갱신
- [ ] 활성 사용자 영향 미리보기 정확
- [ ] rollback 가능 (변경 이력 기반)

#### 🔹 FR-A06: 사용자 지원 도구 (신규)

**Phase**: Phase 4 | **우선순위**: NORMAL

##### 기능
1. **사용자 정지/복구**
   - 정지 사유 입력
   - 사용자에게 이메일 안내
   - 정지 기간 설정 (기간 / 영구)

2. **비밀번호 재설정 안내**
   - 소셜 로그인이라도 계정 잠금 시 재설정 가능
   - 사용자에게 이메일 발송

3. **문의 응대**
