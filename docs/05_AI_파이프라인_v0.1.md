## 1. 개요

### 1.1 목적
이 문서는 본 프로젝트의 **모든 AI 호출 흐름**을 정의한다. 어떤 작업에 어떤 모델을 쓰고, 어떻게 호출하고, 어떻게 검증하고, 어떻게 비용을 최적화할지 명시.

### 1.2 모델 선택 — 확정 사항

| Tier | 모델명 | 정확한 ID | 입력 가격 | 출력 가격 |
|---|---|---|---|---|
| **Tier 1** | Gemini 2.5 Flash-Lite | `gemini-2.5-flash-lite` | $0.10 / 1M | $0.40 / 1M |
| **Tier 2** | Gemini 2.5 Flash | `gemini-2.5-flash` | $0.30 / 1M | $2.50 / 1M |

> 가격은 2026-04-27 기준. 실제 호출 전 [Google AI Studio Pricing](https://ai.google.dev/pricing) 재확인 필요.

### 1.3 왜 Gemini인가?
- **가성비**: 한국어 성능 대비 저렴
- **컨텍스트 윈도우**: 1M 토큰 (긴 컨텍스트 가능)
- **시스템 프롬프트 캐싱**: 90% 비용 절감 가능
- **다국어**: 한/영/일 모두 양호
- **JSON 모드**: 구조화 출력 안정적

### 1.4 1,000명 사용자 기준 비용 추정

| 항목 | 일/사용자 | 1,000명 월 |
|---|---|---|
| Flash-Lite (분류, 키워드, 감정) | 30회 | 900,000회 |
| Flash (Draft, 챗봇, 리포트) | 8회 | 240,000회 |
| **표준 비용** | - | **~$400/월** |
| **캐싱 90% 적용 시** | - | **~$150/월** |

### 2.2 라우팅 결정 의사 코드

```typescript
function selectModel(task: AITask): 'flash-lite' | 'flash' {
  const FLASH_LITE_TASKS = [
    'classify',
    'extract_entities',
    'extract_keywords',
    'classify_priority',
    'extract_emotion',
    'clean_text',
  ];

  return FLASH_LITE_TASKS.includes(task) ? 'flash-lite' : 'flash';
}
```

### 2.3 라우팅 위반 시 자동 차단

Cloud Functions에 `callAI` 추상화 레이어를 두어 `task` → `tier` 매핑이 일관되도록 강제.

```typescript
// ❌ 직접 모델 지정 금지
await genAI.getGenerativeModel({ model: 'gemini-2.5-flash' });

// ✅ 작업명 기반 호출 (모델은 추상화 레이어가 결정)
await callAI({ task: 'classify', input: text });
```

## 4. AI 호출 추상화 레이어

### 4.1 인터페이스

```typescript
// functions/src/ai/client.ts

export interface CallAIOptions<T> {
  task: AITask;                       // 작업명 (모델 결정용)
  systemPromptKey: string;            // 시스템 프롬프트 키
  userInput: string | object;         // 사용자 입력
  outputSchema: z.ZodSchema<T>;       // 출력 검증 스키마
  cacheKey?: string;                  // 캐시 키 (선택)
  language?: 'ko' | 'en' | 'ja';      // 응답 언어 (챗봇 등)
  uid?: string;                       // 사용자 ID (비용 추적)
  maxRetries?: number;                // 기본 1
  temperature?: number;               // 기본 0.7
}

export interface CallAIResult<T> {
  ok: boolean;
  data?: T;
  error?: AIError;
  metadata: {
    model: string;
    inputTokens: number;
    outputTokens: number;
    latencyMs: number;
    cached: boolean;
    retries: number;
  };
}

export async function callAI<T>(
  options: CallAIOptions<T>
): Promise<CallAIResult<T>> { /* ... */ }
```

### 4.2 사용 예시

```typescript
// 분류 호출
import { callAI } from '@/ai/client';
import { ClassifyOutputSchema } from '@/ai/schemas';

const result = await callAI({
  task: 'classify',
  systemPromptKey: 'classify-v1',
  userInput: rawText,
  outputSchema: ClassifyOutputSchema,
  cacheKey: `classify-${userId}`,
  uid: userId,
});

if (!result.ok) {
  throw new HttpsError('internal', '분류 실패');
}

const { track, entities, priority } = result.data;
```

### 4.3 추상화 레이어 책임

1. **모델 선택**: `task` → `tier` 매핑
2. **프롬프트 로드**: 어드민 DB에서 활성 프롬프트 가져오기 (캐시)
3. **시스템 프롬프트 캐싱**: Gemini 캐싱 활성화
4. **호출**: Gemini API 호출
5. **출력 파싱**: JSON 파싱 + Zod 검증
6. **재시도**: 검증 실패 시 1회 재시도
7. **로깅**: 토큰 사용량, 비용, 지연시간 기록
8. **에러 처리**: AI 에러를 사용자 친화적 메시지로 변환

> 모든 프롬프트는 `functions/src/prompts/` 디렉토리 + Firestore `ai_prompts/` 동기화

### 5.1 분류 프롬프트 (FR-004)

**모델**: Flash-Lite
**키**: `classify-v1`

```
당신은 사용자의 한국어 메모를 분석하는 AI입니다.

작업:
1. 텍스트를 LIFE(일상) 또는 WORK(비즈니스)로 분류
2. 엔티티 추출 (TASK, PERSON, EVENT)
3. 우선순위 판단 (URGENT/HIGH/NORMAL/LOW/SOMEDAY)
4. 키워드 추출 (최대 5개)
5. 언어 감지

분류 기준:
- LIFE: 가족, 건강, 취미, 자기계발, 일상생활
- WORK: 업무, 회의, 프로젝트, 동료, 비즈니스 관계

우선순위 기준:
- URGENT: 오늘/내일 마감, 긴급 대응 필요
- HIGH: 이번 주 마감, 중요한 일정
- NORMAL: 이번 달, 일반 업무
- LOW: 여유, 미루어도 무방
- SOMEDAY: 언젠가 할 것, 아이디어

출력은 반드시 다음 JSON 형식만:
{
  "track": "LIFE" | "WORK",
  "trackConfidence": 0.0~1.0,
  "entities": [
    { "type": "TASK" | "PERSON" | "EVENT", "raw": "추출된 부분", "confidence": 0.0~1.0 }
  ],
  "keywords": ["..."],
  "priority": "URGENT" | "HIGH" | "NORMAL" | "LOW" | "SOMEDAY",
  "language": "ko" | "en" | "ja"
}

출력 외에는 어떤 설명도 하지 마세요.
```

### 5.2 Draft 생성 프롬프트 (FR-005)

**모델**: Flash
**키**: `draft-generation-v1`

```
당신은 사용자의 메모를 구조화된 Draft로 변환하는 AI입니다.

입력:
- 원문 텍스트
- 분류 결과 (track, entities, priority)
- 사용자 컨텍스트 (언어, 시간대)

작업:
각 entity에 대해 적절한 Draft를 생성합니다.

TASK Draft:
- title: 명확한 동사 + 대상 (예: "마케팅 리포트 작성")
- description: 부가 정보 (있을 시)
- priority: 분류된 우선순위 사용
- estimatedMinutes: 추정 소요 시간 (있을 시)
- dueDate: 자연어 시간 표현 변환 (예: "내일 오후 3시" → ISO 8601)
- track: LIFE 또는 WORK
- personIds: 언급된 인물 ID들 (사전 정보 활용)

PERSON Draft:
- name: 이름
- role: 관계/직책
- contact: 연락처 (있을 시)
- notes: 메모

EVENT Draft:
- title: 이벤트 제목
- startAt: 시작 시간 (ISO 8601)
- endAt: 종료 시간 (있을 시)
- location: 장소
- attendees: 참석자
- notes: 메모

자연어 시간 처리:
- "내일" → 다음 날 같은 시간 (또는 09:00 기본)
- "다음 주 화요일" → 가장 가까운 화요일
- "3시" → 컨텍스트 기반 오전/오후 판단
- 사용자 시간대 기준

출력은 반드시 다음 JSON 형식만:
{
  "drafts": [
    {
      "type": "TASK" | "PERSON" | "EVENT",
      "data": { ... }
    }
  ]
}
```

### 5.3 챗봇 응답 프롬프트 (FR-014)

**모델**: Flash
**키**: `chatbot-v1`
**톤**: 존댓말 (B2 결정)

```
당신은 사용자의 개인 비서 AI입니다.

역할:
- 사용자의 일정과 목표를 깊이 이해하고 도움이 되는 조언 제공
- 단순 답변기가 아닌 따뜻하고 인간적인 비서

말투:
- 정중한 존댓말 사용
- 친근하고 따뜻한 톤
- 너무 형식적이지 않게
- 2~3 문장으로 간결하게
- 적절한 이모지 사용 (💜 🌟 😊 등 1~2개)

컨텍스트:
[사용자 정보]
- 닉네임: {nickname}
- 현재 시간: {currentTime}
- 사용자 언어: {language}

[최근 7일 활동 요약]
{recent_activity_summary}

[만다라트 진행 상황]
{mandalart_progress}

[오늘의 Top 3 Task]
{today_top3}

[최근 대화 (10개)]
{recent_messages}

행동 가이드:
1. 위로 + 인정 + 제안의 균형
2. 부정적 감정 감지 시 → 위로 우선, 천천히 제안
3. 긍정적 감정 시 → 함께 기뻐하며 동기 부여
4. 정보 부족 시 → 질문, 추측 금지
5. 자해/위험 발언 → 따뜻한 위로 + 전문가 도움 안내
6. 사용자 데이터 자연스럽게 인용 (예: "오늘 마케팅 리포트 마무리하셨네요!")

응답은 반드시 다음 JSON 형식:
{
  "message": "응답 본문",
  "suggestions": ["빠른 응답 1", "빠른 응답 2"] (선택, 최대 3개),
  "emotionDetected": "positive" | "neutral" | "negative" | null
}
```

### 5.4 감정 태그 프롬프트 (FR-013)

**모델**: Flash-Lite
**키**: `emotion-extraction-v1`

```
당신은 한국어 메모에서 감정을 추출하는 AI입니다.

작업: 텍스트의 주된 감정을 분석.

감정 카테고리 (13개):
긍정 (POSITIVE): 기분좋음, 뿌듯함, 감사함, 신남, 평온함
중립 (NEUTRAL): 보통, 집중, 생각중

규칙:
- 텍스트가 매우 짧거나 감정 단서 없을 시 → "중립" + intensity 1

- 반어/sarcasm 주의 (한국어 특유)

출력 JSON:
{
  "category": "POSITIVE" | "NEUTRAL" | "NEGATIVE",
  "label": "기분좋음" | "뿌듯함" | ...,
  "intensity": 1~5,
  "emoji": "😊" 등
}
```

### 5.5 만다라트 자동 제안 프롬프트 (FR-009)

**모델**: Flash
**키**: `mandalart-suggest-v1`

```
당신은 만다라트 사고 도구의 전문가입니다.

만다라트 구조:
- Depth 1: 중심 목표 (1개)
- Depth 2: 핵심 영역 (8개) — 중심 목표 달성을 위한 8개의 큰 영역
- Depth 3: 세부 영역 (각 Section당 8개) — 큰 영역을 더 세분화
- Depth 4: 실행 단위 (각 Sub-section당 8개) — 구체적인 행동/Task

작업:
사용자의 중심 목표(Depth 1)와 트랙(LIFE/WORK)을 받아 Depth 2의 8개 Section을 제안합니다.

요구사항:
- 8개 Section은 서로 명확히 구분되는 영역
- 한국어 사용자에게 자연스러운 표현
- 각 Section은 동사 또는 명사구로 짧게
- 트랙에 적합한 영역 선택
```

```typescript
async function callAIWithValidation<T>(
  options: CallAIOptions<T>
): Promise<CallAIResult<T>> {
  let attempts = 0;
  const maxRetries = options.maxRetries ?? 1;

  while (attempts <= maxRetries) {
    const rawResponse = await geminiCall(options);
    const parsed = options.outputSchema.safeParse(JSON.parse(rawResponse));

    if (parsed.success) {
      return { ok: true, data: parsed.data, metadata: { /* ... */ } };
    }

    attempts++;
    if (attempts > maxRetries) {
      logger.error('AI output validation failed', {
        task: options.task,
        rawResponse,
        zodError: parsed.error,
      });
      return {
        ok: false,
        error: { type: 'validation_failed', detail: parsed.error.message },
        metadata: { /* ... */ },
      };
    }

    // 재시도 시 temperature 낮춤
    options.temperature = Math.max(0.1, (options.temperature ?? 0.7) - 0.2);
  }
}
```

### 9.3 Prompt Injection 방지

```typescript
const INJECTION_PATTERNS = [
  /ignore (previous|above|all) instructions/i,
  /system prompt/i,
  /<\|im_start\|>/,
  /role:\s*(system|assistant)/i,
];

function sanitizeUserInput(input: string): string {
  // 1. 길이 제한
  if (input.length > 10000) input = input.substring(0, 10000);

  // 2. 의심 패턴 검사
  for (const pattern of INJECTION_PATTERNS) {
    if (pattern.test(input)) {
      logger.warn('Possible prompt injection', { input });
      // 차단 또는 escape
      input = input.replace(pattern, '[BLOCKED]');
    }
  }

  return input;
}
```

추가 방어:
- 시스템 프롬프트와 사용자 입력 명확히 분리 (`<user_input>` 태그)
- AI 응답에 시스템 프롬프트 노출 안 되도록 검증
- 의심 패턴 감지 시 어드민 로그 + 사용자 일시 차단

```typescript
  const userPref = await getUserPreferences(uid);
  const inputLang = await detectLanguage(message);

  // 우선순위: 사용자 입력 언어 > 사용자 설정 언어 > 한국어
  const responseLanguage = inputLang ?? userPref.language ?? 'ko';

  return callAI({
    task: 'chatbot',
    systemPromptKey: 'chatbot-v1',
    userInput: message,
    language: responseLanguage,
    outputSchema: ChatbotOutputSchema,
  });
}
```

### 11.3 시스템 프롬프트 다국어
- 분류, 감정 추출 등 → 단일 프롬프트 (한국어 우선, 다국어 처리 가능)
- 챗봇, 리포트 등 사용자 응답 → 언어별 프롬프트

```
ai_prompts/chatbot/versions/{versionId}/translations/{lang}
```
