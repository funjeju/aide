## 0. 문서 개요

본 프로젝트의 사용자 화면 흐름을 정의한다.

**기준 문서**: PRD v0.6 / Firestore 스키마 v0.1 / API Spec v0.1

**다루는 범위:**
- 전체 화면 맵
- 핵심 시나리오 (Phase 1~4)
- 상태 전이도 (Draft, Task, Subscription)
- 에러/한도 초과/오프라인 흐름
- 첫 사용자 온보딩
- 푸시 알림 진입 흐름

## 1. 화면 맵 (전체)

### 1.1 화면 그룹 (Phase별)

#### Phase 1 (Alpha) — 핵심 화면 19개

```
[인증]
  - 로그인
  - 온보딩 (4단계)

[홈]
  - 홈 대시보드 (Top 3 + 오늘 일정)

[입력]
  - 음성 입력
  - 텍스트 입력
  - Draft 검토 (확정 화면)

[ToDo]
  - ToDo 리스트 (LIFE/WORK 탭)
  - Task 상세
  - Task 추가/수정

[만다라트]
  - 프로젝트 리스트
  - 만다라트 보기 (Depth 1~4)
  - 만다라트 자동 제안
  - 만다라트 수동 편집

[캘린더]
  - 캘린더 뷰 (월/주/일)
  - 이벤트 상세

[설정]
  - 설정 메인
  - Google Calendar 연동
  - 알림 설정
  - 계정 설정

[기타]
  - 푸시 알림 진입
  - Empty State 화면
  - 한도 초과 안내
```

#### Phase 2 (Beta 1) — 추가 6개
- 챗봇 메인 (대화 리스트)
- 챗봇 대화 (메시지 화면)
- 감정 타임라인 뷰
- 감정 통계 (간단)
- 비즈니스 리포트 (간소)
- 동기화 상태 / 충돌 해결

#### Phase 3 (Beta 2) — 추가 5개
- 하루 리포트 메인
- 회고 작성
- 인사이트 대시보드
- 테마 선택 / 다크모드 토글
- 반복 Task 설정

#### Phase 4 (정식) — 추가 8개
- Tier 선택 / 결제 / 영수증
- 공유 설정 / 멤버 관리 / 초대 수락
- Export 옵션 / 진행 / 다운로드

### 1.2 네비게이션 구조

#### 모바일 앱 — 5탭 + 중앙 FAB

```
┌──────────────────────────────────────────┐
│           [화면 콘텐츠]                   │
│                                          │
└──────────────────────────────────────────┘
[홈] [할일]  [+]  [캘린더] [설정]
            FAB → 음성/텍스트 입력 모달
```

**탭 구성:**
1. **홈** — 대시보드, Top 3, 오늘 일정
2. **할일** — Task 리스트
3. **+ FAB** (중앙) — 입력 모달 (탭 X, 모달)
4. **캘린더** — 캘린더 뷰
5. **설정** — 메뉴

> 만다라트는 홈에서 카드 진입 또는 ToDo 화면에서 진입.
> 챗봇은 홈 우상단 아이콘 (Phase 2).
> 감정 타임라인은 홈 슬라이드 / Stories 형태 (Phase 2).
> 리포트는 캘린더 화면 상단 메뉴 (Phase 2~3).

## 2. 핵심 시나리오 (Phase 1)

### 2.1 시나리오 1: 첫 사용자 가입 → 첫 입력 (Onboarding)

```
[앱 실행] → [스플래시] (로딩 1초)
   ↓ 미인증 상태
[로그인 화면]
   - "Google로 계속" / "Apple로 계속"
   ↓ 사용자가 Google 선택
[Google OAuth Flow]
   ↓ 인증 성공
[온보딩 1/4: 환영]
   - "○○○에 오신 걸 환영해요"
   - 사용 안내 (애니메이션 1)
   - [다음]
   ↓
[온보딩 2/4: 권한 요청]
   - 마이크 권한 (음성 입력용)
   - 알림 권한 (리마인드용)
   - 캘린더 권한 (Phase 1: 선택, Phase 2: 필수)
   ↓
[온보딩 3/4: 닉네임 입력]
   - "어떻게 불러드릴까요?"
   - [입력 필드]
   ↓
[온보딩 4/4: 알림 시간 설정]
   - "언제 알려드릴까요?"
   - 아침/점심/저녁 시간 선택 (기본값 제공)
   - [완료]
   ↓
[홈 대시보드]
   - Empty State: "첫 입력을 해볼까요?"
   - 중앙 FAB 강조 표시
   ↓ 사용자가 FAB 탭
[입력 모달]
   - [🎤 음성] / [⌨️ 텍스트]
   ↓ 음성 선택
[음성 입력 화면]
   - 마이크 활성화
   - 실시간 STT 표시
   - "오늘 마케팅 리포트 김대리에게 피드백 부탁해야 해"
   ↓ [완료] 버튼 탭
[AI 처리 중] (2~5초)
   - 로딩 애니메이션
   - "AI가 분류하고 있어요..."
   ↓ AI 분류 완료
[Draft 검토 화면]
   - 트랙: WORK
   - Task 1: "마케팅 리포트 피드백" (HIGH)
   - Person: "김대리"
   - 사용자 수정 가능
   ↓ [확정] 버튼
[확정 완료]
   - "Task가 추가되었어요"
   - 토스트 알림
   ↓
[홈 대시보드]
   - Top 3에 새 Task 표시
```

### 2.2 시나리오 2: 음성 입력 → Draft → 확정

> **PRD FR-001 ~ FR-007 통합 흐름**

```
[홈] → [중앙 FAB 탭]
   ↓
[입력 모달]
   - [🎤 음성] / [⌨️ 텍스트] / [📷 카메라(Phase 3)]
   ↓ 음성 선택
[음성 녹음 화면]
   - 마이크 활성화
   - 실시간 STT 텍스트 표시
   - 파형 시각화
   - 한도 체크 (남은 횟수 표시)
   ↓ 사용자 발화 + [완료]
[AI 분류 (Tier 1)]
   - 로딩: "AI가 분류 중..." (1~2초)
   ↓ classifyInput 호출
[AI Draft 생성 (Tier 2)]
   - 로딩: "Draft 생성 중..." (2~3초)
   ↓ generateDraft 호출
[Draft 검토 화면]
   - LIFE/WORK 탭 (자동 분류된 트랙 강조)
   - Task / Person / Event 카드별 표시
   - 각 카드에 [수정] 버튼
   - 신뢰도 < 0.6 시 ⚠️ 표시 (확인 권장)

   사용자 액션:
   1) [확정] — 그대로 저장
   2) [수정] → 인라인 편집
   3) [삭제] — 잘못 분류된 항목 제거
   4) [되돌리기] — 처음부터 다시
   ↓ [확정]
[확정 처리 (confirmDraft 호출)]
   - 한도 체크
   - Task/Person/Event 생성
   - Event인 경우 Google Calendar 등록
   ↓
[성공 화면]
   - "○개 Task, △개 Event가 추가되었어요"
   - 토스트 + 햅틱
   ↓ 자동 닫기
[홈 대시보드]
   - 새 Task가 Top 3에 반영
```

#### Draft 검토 화면 — 빠른 액션
- **수정 빈도가 높은 필드를 메인에**: 제목, 우선순위, 마감일
- **Track 변경**: 상단 토글 (LIFE ↔ WORK)
- **Person 자동 제안**: 기존 Person 매칭 → "김대리(기존)? 새로 만들까요?"

### 2.3 시나리오 3: 만다라트 작성 (자동 제안)

> **PRD FR-009 흐름**

```
[만다라트 진입] → [+ 새 프로젝트]
   ↓
[새 프로젝트 화면]
   - 중심 목표 입력: "건강한 라이프"
   - 트랙: LIFE / WORK 선택
   - [AI 자동 제안] / [직접 작성] 선택
   ↓ AI 자동 제안
[Depth 선택]
   - "어디까지 자동 생성할까요?"
   - [Section만 (8개)] / [Sub-section까지] / [Task까지]
   ↓ Section만 선택
[AI 처리] (3~5초)
   - "만다라트 구조 생성 중..."
   ↓ suggestMandalart 호출
[Section 검토 화면]
   - 8개 Section 카드 (제목 + 이모지)
   - 각 카드: [수정] / [삭제] / [재생성]
   - [전체 다시 생성] / [확정]
   ↓ [확정]
[만다라트 메인 뷰] (Depth 1)
   - 중앙: 목표 "건강한 라이프"
   - 8개 Section (cell)
   - 각 Section 탭 → Depth 2 (Sub-section 8개)
   ↓ Section 탭
[Sub-section 화면] (Depth 2)
   - 중앙: Section 제목
   - 8개 Sub-section
   - 각 Sub-section 탭 → Depth 3 (Task 8개)
   - 미생성 Sub-section: [+ 추가] 또는 [AI 제안]
```

### 2.4 시나리오 4: 홈 → 푸시 알림 → Task 완료

```
[알림 시간 도달] (예: 아침 8:30)
   ↓ FCM 푸시
[푸시 알림 표시]
   - "🌅 좋은 아침이에요! 오늘 우선순위 Top 3"
   ↓ 사용자가 알림 탭
[앱 실행 → 홈 대시보드]
   - Top 3 Task 강조
   - 오늘 일정 표시
   ↓ Task 탭
[Task 상세 화면]
   - 제목, 설명, 우선순위, 마감일
   - 연관 Person, Event
   - [완료 처리] / [편집] / [미루기]
   ↓ [완료 처리] 토글

   - status: 'IN_PROGRESS' → 'DONE'
   - 만다라트 진행률 cascade 갱신
   - 토스트: "잘했어요! 💪"
   ↓ 자동 닫기
[홈 대시보드]
   - Top 3에서 제거, 다음 우선순위 자동 promote
```

## 5. Phase 4 시나리오 (정식)

## 6. 상태 전이도

### 6.1 Draft 상태 전이

```
[입력]
   ↓ classifyInput + generateDraft
[PENDING] (Draft)
   ↓ 사용자 액션
   ├─ [확정] → CONFIRMED → Task/Event/Person 생성
   ├─ [수정 후 확정] → 같은 흐름
   ├─ [삭제] → DELETED (휴지통, 7일 보관)
   └─ [방치] → 7일 후 자동 DELETED

[DELETED]
   ↓ 7일 후
[영구 삭제]
```

### 6.2 Task 상태 전이

```
[Draft 확정]
   ↓
[CONFIRMED]
   ↓ 사용자가 시작
[IN_PROGRESS]
   ↓
   ├─ [완료] → DONE → 만다라트 진행률 갱신
   ├─ [취소] → CANCELED
   └─ [미루기] → CONFIRMED + dueDate 변경

[DONE]
   ↓ 사용자가 되돌리기 (1시간 이내)
[IN_PROGRESS]
```

### 6.3 Subscription 상태 전이 (Phase 4)

```
[Free 사용자]
   ↓ 결제 (verifyApplePurchase)
[active] (BASIC/PRO/ENT)
   ↓ 자동 갱신 끄기
[active + autoRenew: false]
   ↓ 만료일 도달
[expired] → Free 강등

[active]
   ↓ 결제 실패
[grace_period] (3일)
   ↓ 결제 성공
   ├─ [active] 복원
   └─ [expired] 결제 실패 지속

[expired]
   ↓ 다시 결제
[active]
```

### 6.4 Share 상태 전이 (Phase 4)

```
[초대 발송]
   ↓
[pending]
   ↓ 사용자 액션
   ├─ [accepted] → 접근 가능
   ├─ [declined] → 표시 X
   └─ [방치] → 30일 후 자동 만료

[accepted]
   ↓ Owner가 제거
[removed]
```

## 10. 핵심 디자인 원칙

### 10.1 인지 부하 최소화
- 메인 화면 정보는 **Top 3 + 오늘 일정**만 (스크롤 불필요)
- 부가 정보는 진입 후 노출

### 10.2 입력 최소화
- 음성 입력 우선 (한 번 누르면 끝)
- 자동 분류 (사용자가 LIFE/WORK 직접 선택할 필요 없음)
- AI 추천 우선 (사용자는 검토만)

### 10.3 작업 흐름 단축
- 가장 흔한 작업은 1-2탭으로
  - "Task 완료" = 1탭
  - "음성 입력 → Draft 확정" = 3탭
- 부가 작업은 더 깊이 (만다라트 편집 등)

### 10.4 시각적 일관성
- LIFE = 그린 (#10B981) / WORK = 블루 (#3B82F6)
- 우선순위 색상 일관 (URGENT 빨강 → SOMEDAY 회색)
- Track 표시 항상 좌상단

### 10.5 즉각적 피드백
- 햅틱 피드백 (확정/완료 시)
- 토스트 알림 (성공/에러)
- 로딩 상태 명확 (스켈레톤 + 진행 안내)
