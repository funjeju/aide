## 2. 결제 인프라

### 2.1 RevenueCat (강력 권장)

**왜 RevenueCat?**
- iOS + Android 통합 처리
- 영수증 검증 자동
- 환불/취소 webhook 자동
- 분석 대시보드 (LTV, churn)
- 무료 플랜으로 시작 가능 (월 매출 $10K까지)

### 2.2 결제 플랫폼

| 플랫폼 | 결제 방식 | 수수료 |
|---|---|---|
| **iOS** | Apple In-App Purchase + StoreKit 2 | 30% (1년 후 15%) |
| **Android** | Google Play Billing | 30% (1년 후 15%) |
| **Web (선택)** | Stripe | ~3% |

**웹 결제 정책**:
- iOS 앱에서 외부 결제 유도 → Apple 정책 위반
- 웹 가입 페이지(별도 도메인)에서만 Stripe 결제 안내
- Pro 이상에서만 웹 결제 옵션

## 3. 구독 라이프사이클

### 3.1 구독 시작

```
사용자 한도 초과 → 업그레이드 모달
    ↓
요금제 화면 → Tier 선택 → 월/년 선택
    ↓
무료 체험 옵션 (Pro만):
    [14일 무료 체험] → 카드 등록 X → 즉시 Pro 활성
    [지금 구독] → IAP 결제 시트
    ↓
[결제 성공]
    ↓
verifyApplePurchase / verifyGooglePurchase Cloud Function
    ↓
영수증 검증 → 위조 차단
    ↓
subscriptions/ 새 문서 생성
    ↓
users/{uid}.userTier 업데이트
    ↓
환영 이메일 발송
    ↓
"Pro에 오신 것을 환영해요!" 화면
```

### 3.2 자동 갱신
- IAP 기본 동작 (사용자 명시적 취소 전까지)
- 만료 24시간 전 자동 갱신
- RevenueCat webhook 수신 → `subscriptions.endDate` 연장

### 3.3 갱신 실패 (Grace Period)
- Apple/Google이 자동으로 7일 grace period 부여
- 이 기간 동안 사용자에게 이메일 + 인앱 알림
- 7일 내 결제 정보 업데이트 → 갱신
- 7일 경과 → 자동 Free 강등

### 3.4 업그레이드 (예: Basic → Pro)
- 사용자가 Pro 선택 → IAP 처리
- Apple/Google이 일할 차액 처리
- 기존 Basic 구독 자동 종료, Pro 새로 시작

### 3.5 다운그레이드 (예: Pro → Basic)
- 사용자가 다운그레이드 신청
- 현재 결제 기간 동안은 Pro 유지
- 다음 갱신 시점에 Basic으로 변경
- IAP는 자동 처리

### 3.6 취소 (Cancel)
- 사용자가 자동 갱신 끄기
- IAP 시스템 다이얼로그로 안내
- 결제 기간 동안 유지 → 만료 후 Free 강등

### 3.7 환불

**자동 환불 조건** (자체 정책):
- 구매 후 7일 이내
- AI 호출 < 5회 (사용 거의 없음)
- → 자동 처리

**수동 환불**:
- 위 조건 미달 → 어드민 검토
- 24시간 이내 응답

**Apple/Google 환불**:
- 사용자가 직접 Apple/Google에 요청
- Webhook 수신 → 자동 강등 처리

## 5. RevenueCat 통합

### 5.1 셋업
1. RevenueCat 계정 생성
2. iOS / Android 앱 등록
3. App Store Connect / Google Play Console에서 제품 생성:
   - `basic_monthly` / `basic_yearly`
   - `pro_monthly` / `pro_yearly`
   - `enterprise_monthly` / `enterprise_yearly`
4. RevenueCat에서 Entitlements 설정 (`pro_features` 등)

### 5.2 클라이언트 통합

```typescript
// apps/mobile/src/features/payment/services/purchases.ts
import Purchases from 'react-native-purchases';

export async function initPurchases(uid: string) {
  await Purchases.configure({
    apiKey: process.env.EXPO_PUBLIC_REVENUECAT_IOS_KEY!,
    appUserID: uid,
  });
}

export async function purchasePackage(packageId: string) {
  const offerings = await Purchases.getOfferings();
  const pkg = offerings.current?.availablePackages.find(p => p.identifier === packageId);
  if (!pkg) throw new Error('Package not found');

  const { customerInfo } = await Purchases.purchasePackage(pkg);
  return customerInfo.entitlements.active;
}

export async function restorePurchases() {
  const customerInfo = await Purchases.restorePurchases();
  return customerInfo.entitlements.active;
}
```

### 5.3 서버 측 webhook

```typescript
// functions/src/http/webhooks/revenueCat.ts
export const revenueCatWebhook = onRequest(
  { region: 'asia-northeast3' },
  async (req, res) => {
    // 1. 서명 검증
    const signature = req.headers['authorization'];
    if (!verifyRevenueCatSignature(signature, req.body)) {
      res.status(401).send('Invalid signature');
      return;
    }

    const { event } = req.body;

    // 2. 이벤트 처리
    switch (event.type) {
      case 'INITIAL_PURCHASE':
        await handleInitialPurchase(event);
        break;
      case 'RENEWAL':
        await handleRenewal(event);
        break;
      case 'CANCELLATION':
        await handleCancellation(event);
        break;
      case 'EXPIRATION':
        await handleExpiration(event);
        break;
      case 'BILLING_ISSUE':
        await handleBillingIssue(event);
        break;
      case 'REFUND':
        await handleRefund(event);
        break;
    }

    res.status(200).send('OK');
  }
);
```

## 10. 환불 / 분쟁 처리

### 10.1 환불 흐름
1. 사용자 환불 요청 (앱 내)
2. 자동 환불 조건 체크
3. 조건 미달 → 어드민 큐
4. 어드민이 검토 → 승인/거절
5. 승인 시 Apple/Google API 통해 환불
6. Webhook 수신 → 자동 강등

### 10.2 chargeback
- 신용카드사 chargeback (분쟁) → Apple/Google이 1차 대응
- 어드민에게 알림 → 사용자 정지 검토
